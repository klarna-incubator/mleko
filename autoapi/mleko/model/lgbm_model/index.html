Wrapper for the LightGBM model.

Full documentation of the LightGBM model can be found here https://lightgbm.readthedocs.io/en/latest/index.html.

Initialize the LightGBM model with the given hyperparameters.

Note

Features and ignore_features are mutually exclusive. If both are provided, a ValueError will be raised. By default, all features are used. If ignore_features is provided, all features except the ones in ignore_features will be used. If features is provided, only the features in features will be used.

Parameters:

  • target (str) – The name of the target feature.

  • feval (lightgbm.engine._LGBM_CustomMetricFunction | list[lightgbm.engine._LGBM_CustomMetricFunction] | tuple[lightgbm.engine._LGBM_CustomMetricFunction, Ellipsis] | None) – Custom evaluation function(s). Should return a tuple (eval_name, eval_result, is_higher_better). Refer to https://lightgbm.readthedocs.io/en/latest/pythonapi/lightgbm.train.html#lightgbm.train.

  • features (list[str] | tuple[str, Ellipsis] | None) – The names of the features to be used as input for the model.

  • ignore_features (list[str] | tuple[str, Ellipsis] | None) – The names of the features to be ignored.

  • objective (LGBMObjectiveType) – The objective function to be used.

  • boosting (LGBMBoostingType) – The type of boosting algorithm.

  • data_sample_strategy (LGBMDataSampleStrategyType) – How data sampling should be performed.

  • num_iterations (int) – The number of iterations to perform.

  • learning_rate (float) – The learning rate.

  • num_leaves (int) – The maximum number of leaves in one tree.

  • tree_learner (LGBMTreeLearnerType) – The type of tree learner.

  • random_state (int | None) – The seed to use for random number generation.

  • force_col_wise (bool) – Force col-wise histogram building.

  • force_row_wise (bool) – Force row-wise histogram building.

  • histogram_pool_size (float) – Size of pooled memory for histograms in MB.

  • max_depth (int) – The maximum depth of a tree.

  • min_data_in_leaf (int) – The minimum number of data in one leaf.

  • min_sum_hessian_in_leaf (float) – The minimum sum of hessian in one leaf.

  • bagging_fraction (float) – The fraction of data to be used when bagging.

  • pos_bagging_fraction (float) – The fraction of positive data to be used when bagging, only used in binary application.

  • neg_bagging_fraction (float) – The fraction of negative data to be used when bagging, only used in binary application.

  • bagging_freq (int) – Frequency for bagging, 0 means disable bagging; k means perform bagging at every k iteration. At every bagging iteration, LightGBM will randomly select baggig_fraction * data_size data from training data to train for the next k iterations.

  • feature_fraction (float) – The fraction of features to be used when building each tree.

  • feature_fraction_bynode (float) – The fraction of features to be used for each tree node.

  • extra_trees (bool) – Whether to use extremely randomized trees.

  • early_stopping_round (int | None) – Will stop training if one metric of one validation data doesn’t improve in last early_stopping_round rounds.

  • first_metric_only (bool) – Whether to only use the first metric for early stopping.

  • max_delta_step (float) – The maximum delta step for tree leaf output.

  • lambda_l1 (float) – L1 regularization.

  • lambda_l2 (float) – L2 regularization.

  • linear_lambda (float) – Linear tree regularization.

  • min_gain_to_split (float) – The minimal gain to perform split.

  • drop_rate (float) – Dropout rate, only used in dart boosting, will drop trees with probability drop_rate.

  • max_drop (int) – Maximum number of dropped trees during one iteration, only used in dart boosting.

  • skip_drop (float) – Probability of skipping the dropout procedure during one iteration, only used in dart boosting.

  • xgboost_dart_mode (bool) – Whether to use xgboost dart boosting.

  • uniform_drop (bool) – Whether to use uniform drop, only used in dart boosting.

  • top_rate (float) – The retain ratio of large gradient data, only used in goss boosting.

  • other_rate (float) – The retain ratio of small gradient data, only used in goss boosting.

  • min_data_per_group (int) – The minimum number of data per categorical group.

  • max_cat_threshold (int) – Limit number of split points for categorical features.

  • cat_l2 (float) – L2 regularization in categorical split.

  • cat_smooth (float) – Can reduce the effect of noises in categorical features, especially for categories with few data.

  • max_cat_to_onehot (int) – When number of categories of one feature smaller than or equal to max_cat_to_onehot, one-vs-other split algorithm will be used.

  • top_k (int) – Set this to larger value for more accurate result, but it will slow down the training speed.

  • monotone_constraints (list[int] | tuple[int, Ellipsis] | None) – The constraints for monotonicity, set it to a list with the same size as the number of features to turn on the monotonicity constraint. 1 means increasing constraint, -1 means decreasing constraint, and 0 means no constraint.

  • monotone_constraints_method (LGBMMonotoneConstraintsMethodType) – The method to use to deal with monotone constraints, only used when monotone_constraints is not None.

  • monotone_penalty (float) – The penalty for monotone constraints.

  • feature_contri (list[float] | tuple[float, Ellipsis] | None) – Used to control feature’s split gain, will use gain[i] = max(0, feature_contri[i]) * gain[i] to replace the original gain of the ith feature.

  • path_smooth (float) – Controls the smoothing applied to tree nodes.

  • verbosity (int) – Controls the level of LightGBM’s verbosity.

  • log_evaluation_period (int | None) – The period to log the evaluation results, if None, results will not be logged.

  • use_quantized_grad (bool) – Whether to use gradiant quantization when training.

  • num_grad_quant_bins (int) – Number of bin to quantization gradients and hessians.

  • quant_train_renew_leaf (bool) – Whether to renew the leaf values with original gradients when quantized training.

  • stochastic_rounding (bool) – Whether to use stochastic rounding in gradient quantization.

  • linear_tree (bool) – Fit piecewise linear gradient boosting tree.

  • max_bin (int) – Max number of bins that feature values will be bucketed in.

  • max_bin_by_feature (list[int] | tuple[int, Ellipsis] | None) – Max number of bins that feature values will be bucketed in, per feature.

  • min_data_in_bin (int) – Min number of data inside one bin.

  • bin_construct_sample_cnt (int) – Number of data that sampled to construct histogram bins.

  • is_enable_sparse (bool) – Used to enable/disable the sparse optimization.

  • enable_bundle (bool) – Set this to False to disable Exclusive Feature Bundling (EFB).

  • use_missing (bool) – Set this to False to disable the special handle of missing value.

  • zero_as_missing (bool) – Set this to True to treat zero as missing value.

  • feature_pre_filter (bool) – Set this to True to ignroe the features that are unsplittable based on min_data_in_leaf.

  • num_class (int) – Only used in multiclass classification.

  • is_unbalance (bool) – Used only in binary and multiclassova classification application, used to handle unbalanced training data. Mutually exclusive with scale_pos_weight.

  • scale_pos_weight (float) – Used only in binary and multiclassova classification application, specifies the ratio of positive class weight to negative class weight. Mutually exclusive with is_unbalance.

  • sigmoid (float) – Used only in binary and multiclassova classification and in lambdarank application, specifies the sigmoid parameter.

  • boost_from_average (bool) – Used only in regression, binary, multiclassova and cross-entropy applications, adjusts initial score to the mean of labels for faster convergence.

  • reg_sqrt (bool) – Used only in regression application, specifies whether to use sqrt(label) as labels.

  • alpha (float) – Used only in huber and quantile regression application, specifies the alpha parameter.

  • fair_c (float) – Used only in fair regression application, specifies the parameter for Fair loss.

  • poisson_max_delta_step (float) – Used only in poisson regression application, specifies the parameter for Poisson regression to safeguard optimization.

  • tweedie_variance_power (float) – Used only in tweedie regression application. Used to control the variance of the Tweedie distribution. Set closer to 2 to shift towards a Gamma distribution. Set closer to 1 to shift towards a Poisson distribution.

  • lambdarank_truncation_level (int) – Used only in lambdarank application, controls the number of top-results to focus on during training.

  • lambdarank_norm (bool) – Set this to True to normalize the lambdas for different queries, and improve the performance for unbalanced data. Set this to False to enforce the original lambdarank algorithm. Used only in lambdarank application.

  • metric (LGBMMetricType | list[LGBMMetricType]) – Metrics to be evaluated on the evaluation set(s).

  • metric_freq (int) – The frequency of metric output, every metric_freq iterations.

  • is_provide_training_metric (bool) – Set this to True to output the metric result over training data.

  • eval_at (list[int] | tuple[int, Ellipsis] | None) – Used only with ndcg and map metrics, specifies the evaluation positions.

  • multi_error_top_k (int) – Used only in multi_error metric, specifies the threshold for top-k error metric.

  • auc_mu_weights (list[float] | tuple[float, Ellipsis] | None) – Used only in auc_mu metric, specifies the weights for auc_mu metric.

  • cache_directory (str | pathlib.Path) – The target directory where the model will be saved.

  • cache_size (int) – The maximum number of entries to keep in the cache.

Examples

>>> import vaex
>>> from mleko.model import LGBMModel
>>> df = vaex.ml.datasets.load_iris()
>>> df_train, df_test = df.ml.random_split(test_size=0.20, verbose=False)
>>> data_schema = DataSchema(
...    numerical=["sepal_length", "sepal_width", "petal_length", "petal_width"],
... )
>>> model = LGBMModel(
...     target="class_",
...     features=["sepal_width", "petal_length", "petal_width"],
...     num_iterations=100,
...     learning_rate=0.1,
...     num_leaves=31,
...     random_state=42,
... )
>>> booster, df_train_pred, df_test_pred = model.fit_transform(data_schema, df_train, df_test, {})
