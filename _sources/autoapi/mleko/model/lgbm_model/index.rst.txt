   Wrapper for the LightGBM model.

   Full documentation of the LightGBM model can be found here https://lightgbm.readthedocs.io/en/latest/index.html.

   Initialize the LightGBM model with the given hyperparameters.

   .. note::

      Features and ignore_features are mutually exclusive. If both are provided, a `ValueError` will be raised.
      By default, all features are used. If ignore_features is provided, all features except the ones in
      ignore_features will be used. If features is provided, only the features in features will be used.

   :param target: The name of the target feature.
   :param feval: Custom evaluation function(s). Should return a tuple (eval_name, eval_result, is_higher_better).
                 Refer to https://lightgbm.readthedocs.io/en/latest/pythonapi/lightgbm.train.html#lightgbm.train.
   :param features: The names of the features to be used as input for the model.
   :param ignore_features: The names of the features to be ignored.
   :param objective: The objective function to be used.
   :param boosting: The type of boosting algorithm.
   :param data_sample_strategy: How data sampling should be performed.
   :param num_iterations: The number of iterations to perform.
   :param learning_rate: The learning rate.
   :param num_leaves: The maximum number of leaves in one tree.
   :param tree_learner: The type of tree learner.
   :param random_state: The seed to use for random number generation.
   :param force_col_wise: Force col-wise histogram building.
   :param force_row_wise: Force row-wise histogram building.
   :param histogram_pool_size: Size of pooled memory for histograms in MB.
   :param max_depth: The maximum depth of a tree.
   :param min_data_in_leaf: The minimum number of data in one leaf.
   :param min_sum_hessian_in_leaf: The minimum sum of hessian in one leaf.
   :param bagging_fraction: The fraction of data to be used when bagging.
   :param pos_bagging_fraction: The fraction of positive data to be used when bagging,
                                only used in binary application.
   :param neg_bagging_fraction: The fraction of negative data to be used when bagging,
                                only used in binary application.
   :param bagging_freq: Frequency for bagging, 0 means disable bagging; k means perform bagging at every k iteration.
                        At every bagging iteration, LightGBM will randomly select `baggig_fraction` * `data_size` data from
                        training data to train for the next `k` iterations.
   :param feature_fraction: The fraction of features to be used when building each tree.
   :param feature_fraction_bynode: The fraction of features to be used for each tree node.
   :param extra_trees: Whether to use extremely randomized trees.
   :param early_stopping_round: Will stop training if one metric of one validation data doesn't improve in last
                                `early_stopping_round` rounds.
   :param first_metric_only: Whether to only use the first metric for early stopping.
   :param max_delta_step: The maximum delta step for tree leaf output.
   :param lambda_l1: L1 regularization.
   :param lambda_l2: L2 regularization.
   :param linear_lambda: Linear tree regularization.
   :param min_gain_to_split: The minimal gain to perform split.
   :param drop_rate: Dropout rate, only used in `dart` boosting, will drop trees with probability `drop_rate`.
   :param max_drop: Maximum number of dropped trees during one iteration, only used in `dart` boosting.
   :param skip_drop: Probability of skipping the dropout procedure during one iteration, only
                     used in `dart` boosting.
   :param xgboost_dart_mode: Whether to use xgboost `dart` boosting.
   :param uniform_drop: Whether to use uniform drop, only used in `dart` boosting.
   :param top_rate: The retain ratio of large gradient data, only used in `goss` boosting.
   :param other_rate: The retain ratio of small gradient data, only used in `goss` boosting.
   :param min_data_per_group: The minimum number of data per categorical group.
   :param max_cat_threshold: Limit number of split points for categorical features.
   :param cat_l2: L2 regularization in categorical split.
   :param cat_smooth: Can reduce the effect of noises in categorical features, especially for
                      categories with few data.
   :param max_cat_to_onehot: When number of categories of one feature smaller than or equal to `max_cat_to_onehot`,
                             one-vs-other split algorithm will be used.
   :param top_k: Set this to larger value for more accurate result, but it will slow down the training speed.
   :param monotone_constraints: The constraints for monotonicity, set it to a list with the same size as the number
                                of features to turn on the monotonicity constraint. `1` means increasing constraint, `-1` means
                                decreasing constraint, and `0` means no constraint.
   :param monotone_constraints_method: The method to use to deal with monotone constraints, only used when
                                       `monotone_constraints` is not None.
   :param monotone_penalty: The penalty for monotone constraints.
   :param feature_contri: Used to control feature's split gain, will
                          use gain[i] = max(0, feature_contri[i]) * gain[i] to replace the original gain of the ith feature.
   :param path_smooth: Controls the smoothing applied to tree nodes.
   :param verbosity: Controls the level of LightGBM's verbosity.
   :param log_evaluation_period: The period to log the evaluation results, if `None`, results will not be logged.
   :param use_quantized_grad: Whether to use gradiant quantization when training.
   :param num_grad_quant_bins: Number of bin to quantization gradients and hessians.
   :param quant_train_renew_leaf: Whether to renew the leaf values with original gradients when quantized training.
   :param stochastic_rounding: Whether to use stochastic rounding in gradient quantization.
   :param linear_tree: Fit piecewise linear gradient boosting tree.
   :param max_bin: Max number of bins that feature values will be bucketed in.
   :param max_bin_by_feature: Max number of bins that feature values will be bucketed in, per feature.
   :param min_data_in_bin: Min number of data inside one bin.
   :param bin_construct_sample_cnt: Number of data that sampled to construct histogram bins.
   :param is_enable_sparse: Used to enable/disable the sparse optimization.
   :param enable_bundle: Set this to `False` to disable Exclusive Feature Bundling (EFB).
   :param use_missing: Set this to `False` to disable the special handle of missing value.
   :param zero_as_missing: Set this to `True` to treat zero as missing value.
   :param feature_pre_filter: Set this to `True` to ignroe the features that are unsplittable based
                              on `min_data_in_leaf`.
   :param num_class: Only used in multiclass classification.
   :param is_unbalance: Used only in `binary` and `multiclassova` classification application, used
                        to handle unbalanced training data. Mutually exclusive with `scale_pos_weight`.
   :param scale_pos_weight: Used only in `binary` and `multiclassova` classification application, specifies the
                            ratio of positive class weight to negative class weight. Mutually exclusive with `is_unbalance`.
   :param sigmoid: Used only in `binary` and `multiclassova` classification and in `lambdarank` application,
                   specifies the sigmoid parameter.
   :param boost_from_average: Used only in `regression`, `binary`, `multiclassova` and `cross-entropy` applications,
                              adjusts initial score to the mean of labels for faster convergence.
   :param reg_sqrt: Used only in `regression` application, specifies whether to use `sqrt(label)` as labels.
   :param alpha: Used only in `huber` and `quantile` `regression` application, specifies the alpha parameter.
   :param fair_c: Used only in `fair` `regression` application, specifies the parameter for Fair loss.
   :param poisson_max_delta_step: Used only in `poisson` `regression` application, specifies the parameter
                                  for Poisson regression to safeguard optimization.
   :param tweedie_variance_power: Used only in `tweedie` `regression` application. Used to control the variance of
                                  the Tweedie distribution. Set closer to 2 to shift towards a Gamma distribution. Set closer to 1 to
                                  shift towards a Poisson distribution.
   :param lambdarank_truncation_level: Used only in `lambdarank` application, controls the number of top-results
                                       to focus on during training.
   :param lambdarank_norm: Set this to `True` to normalize the lambdas for different queries, and improve the
                           performance for unbalanced data. Set this to `False` to enforce the original lambdarank algorithm.
                           Used only in `lambdarank` application.
   :param metric: Metrics to be evaluated on the evaluation set(s).
   :param metric_freq: The frequency of metric output, every `metric_freq` iterations.
   :param is_provide_training_metric: Set this to `True` to output the metric result over training data.
   :param eval_at: Used only with `ndcg` and `map` metrics, specifies the evaluation positions.
   :param multi_error_top_k: Used only in `multi_error` metric, specifies the threshold for top-k error metric.
   :param auc_mu_weights: Used only in `auc_mu` metric, specifies the weights for `auc_mu` metric.
   :param cache_directory: The target directory where the model will be saved.
   :param cache_size: The maximum number of entries to keep in the cache.

   .. rubric:: Examples

   >>> import vaex
   >>> from mleko.model import LGBMModel
   >>> df = vaex.ml.datasets.load_iris()
   >>> df_train, df_test = df.ml.random_split(test_size=0.20, verbose=False)
   >>> data_schema = DataSchema(
   ...    numerical=["sepal_length", "sepal_width", "petal_length", "petal_width"],
   ... )
   >>> model = LGBMModel(
   ...     target="class_",
   ...     features=["sepal_width", "petal_length", "petal_width"],
   ...     num_iterations=100,
   ...     learning_rate=0.1,
   ...     num_leaves=31,
   ...     random_state=42,
   ... )
   >>> booster, df_train_pred, df_test_pred = model.fit_transform(data_schema, df_train, df_test, {})

   .. py:method:: _pop_num_iterations(hyperparameters: dict) -> int

      Extracts the number of iterations from the hyperparameters.

      .. warning::

         All aliases for the number of iterations are removed from the hyperparameters dictionary.
         This is an intentional side effect which is used to avoid passing the number of iterations
         to the LightGBM model twice.

      :param hyperparameters: The hyperparameters dictionary.

      :returns: The number of iterations.
